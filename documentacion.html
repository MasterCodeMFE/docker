Comando Descripción Opciones
docker stats Muestra uso de CPU, RAM y otros datos de consumo de cada contenedor activo.
--format 'table' : Formato por defecto es 'table'.
--format 'json' : Cambia el formato a 'json'.
--no-stream : Muestra una única instantánea en lugar de un flujo continuo.

docker pull <imagen> Descarga una imagen de Docker desde un registro (como Docker Hub).
-a, --all-tags : Descarga todas las etiquetas de la imagen.

docker run <opciones> <imagen> Crea un contenedor a partir de una imagen y lo ejecuta.
-d, --detach : Ejecuta en segundo plano.
--name <nombre> : Asigna un nombre al contenedor.
-it : Interactivo con terminal.
-p <puerto_host>:<puerto_contenedor> : Mapea puertos.
-e <VARIABLE=valor> : Establece variables de entorno.
--rm : Elimina el contenedor al detenerlo.
--restart <policy> : Política de reinicio (always, unless-stopped, etc.).
--network <red> : Conecta el contenedor a una red específica.

-a, --all : Muestra todos los contenedores (activos y detenidos).
-q, --quiet : Muestra solo los IDs de los contenedores.
--no-trunc : No trunca el texto de salida.
--filter <filtro> : Filtra resultados según ciertos criterios.

-q : Muestra solo los IDs de las imágenes.
--digests : Muestra los resúmenes de las imágenes.
--no-trunc : No trunca la salida.
--filter <filtro> : Filtra imágenes según criterios específicos.

-f, --force : Fuerza la eliminación de contenedores en ejecución.
-v, --volumes : Elimina volúmenes asociados.

-f, --force : Fuerza la eliminación.
--no-prune : No elimina imágenes no referenciadas.

-t <segundos> : Espera un tiempo específico antes de forzar la detención.

-f, --follow : Sigue los registros en tiempo real.
--tail <n> : Muestra solo las últimas <n> líneas.
--timestamps : Muestra marcas de tiempo.

-d, --detach : Ejecuta el comando en segundo plano.
-i, --interactive : Mantiene la entrada estándar abierta.
-t, --tty : Asigna un terminal virtual.
docker build <opciones> <ruta> Construye una imagen de Docker a partir de un archivo Dockerfile y un contexto.
--no-cache : No utiliza la caché de las capas anteriores.
-f <Dockerfile> : Especifica un Dockerfile alternativo.
--build-arg <arg>=<valor> : Establece argumentos de construcción.
-q, --quiet : Silencia la salida.
docker image pull <imagen> Descarga una imagen desde un registro. Ninguna
docker image ls Lista imágenes en tu sistema.
-q : Muestra solo los IDs de las imágenes.
--digests : Muestra los resúmenes de las imágenes.
--no-trunc : No trunca la salida.
docker image rm <imagen> Elimina una o más imágenes.
-f, --force : Fuerza la eliminación.
--no-prune : No elimina imágenes no referenciadas.
docker image prune Elimina imágenes no utilizadas.
-a, --all : Elimina todas las imágenes no utilizadas, no solo las "dangling".
--filter <filter> : Aplica filtros.

-q, --quiet : Muestra solo los IDs de los contenedores.
docker container rm <contenedor> Elimina uno o más contenedores.
-f, --force : Fuerza la eliminación de contenedores en ejecución.
-v, --volumes : Elimina volúmenes asociados.
docker container prune Elimina contenedores detenidos.
--filter <filter> : Aplica filtros.
docker container inspect <contenedor> Muestra información detallada sobre un contenedor. Ninguna
docker network ls Lista las redes de Docker en tu sistema.
-q, --quiet : Muestra solo los IDs de las redes.
--filter <filter> : Aplica filtros.

--driver <controlador> : Especifica el controlador de la red (bridge, host, overlay, etc.).
--subnet <subred> : Especifica la subred de la red.
--gateway <puerta_de_enlace> : Especifica la puerta de enlace de la red.

docker network disconnect <red> <contenedor> Desconecta un contenedor de una red. Ninguna
docker volume ls Lista los volúmenes de Docker en tu sistema.
-q, --quiet : Muestra solo los IDs de los volúmenes.
--filter <filter> : Aplica filtros.

docker compose up Inicia contenedores definidos en un archivo docker-compose.yml.
-d, --detach : Ejecuta en segundo plano.
--build : Fuerza la reconstrucción de las imágenes.
--force-recreate : Fuerza la recreación de los contenedores.
--remove-orphans : Elimina contenedores huérfanos.
-p <nombre> : Personaliza el nombre del stack.
docker compose down Detiene y elimina contenedores definidos en un archivo docker-compose.yml.
-v, --volumes : Elimina volúmenes asociados.
--remove-orphans : Elimina contenedores huérfanos.
--rmi <all|local> : Elimina imágenes según el criterio especificado.
docker compose logs Muestra los registros de contenedores definidos en un archivo docker-compose.yml.
-f, --follow : Sigue los registros en tiempo real.
--tail <n> : Muestra solo las últimas <n> líneas.
docker info Muestra información sobre la instalación de Docker. Ninguna
docker version Muestra la versión de Docker que está instalada. Ninguna
docker inspect <objeto> Muestra información detallada sobre un objeto Docker (imagen, contenedor, volumen, red, etc.). Ninguna
docker system prune Limpia recursos no utilizados.
-a, --all : Elimina imágenes, contenedores y redes no utilizados.
--volumes : También elimina volúmenes no utilizados.
docker builder prune Limpia componentes de construcción, incluyendo imágenes intermedias.
--all : Elimina todas las imágenes intermedias no referenciadas.
-f, --force : No pide confirmación antes de eliminar.

Guía Completa de Uso de Docker

1. Introducción a Docker

Docker es una plataforma que permite a los desarrolladores empaquetar aplicaciones y sus dependencias en contenedores. Un contenedor es una unidad estándar de software que empaqueta el código y todas sus dependencias para que la aplicación se ejecute rápidamente y de manera confiable en diferentes entornos de computación. Docker te permite crear, desplegar y ejecutar aplicaciones en contenedores, lo que mejora la eficiencia y la consistencia en el desarrollo y producción.

2. Instalación de Docker

Para instalar Docker, sigue las instrucciones específicas para tu sistema operativo en la documentación oficial de Docker. Asegúrate de que tu sistema cumpla con los requisitos de instalación, que pueden variar según el sistema operativo.

  • Windows y Mac: Descarga Docker Desktop desde el sitio oficial y sigue las instrucciones de instalación.
  • Linux: Utiliza el gestor de paquetes de tu distribución. Por ejemplo, en Ubuntu, puedes usar: 
    sudo apt update
    sudo apt install docker.io
    sudo systemctl start docker
    sudo systemctl enable docker
    sudo usermod -aG docker $USER

3. Conceptos Básicos

  • Contenedor: Una instancia de una imagen Docker. Los contenedores son ligeros y se pueden ejecutar en cualquier entorno que soporte Docker.
  • Imagen: Un paquete que contiene todo lo necesario para ejecutar una aplicación, incluyendo el código, las bibliotecas y las dependencias. Las imágenes son inmutables.
  • Dockerfile: Un archivo de texto que contiene instrucciones para construir una imagen Docker. Las instrucciones son ejecutadas por Docker para crear una imagen.
  • Docker Hub: Un servicio de registro en línea que permite almacenar y compartir imágenes Docker. Puedes subir tus propias imágenes o descargar imágenes públicas.
  • Docker Compose: Una herramienta para definir y ejecutar aplicaciones Docker de múltiples contenedores utilizando un archivo YAML. Permite gestionar la configuración y los servicios de forma sencilla.
  • Volumen: Un mecanismo para persistir datos generados y utilizados por contenedores. Los volúmenes permiten que los datos sobrevivan a los ciclos de vida de los contenedores.
  • Redes: Docker permite crear redes personalizadas para conectar contenedores. Esto proporciona aislamiento y control sobre la comunicación entre contenedores.

4. Comandos Comunes

Algunos comandos esenciales para trabajar con Docker son:

  • docker pull: Descarga imágenes desde Docker Hub.
  • docker run: Crea y ejecuta un nuevo contenedor.
  • docker ps: Lista los contenedores en ejecución.
  • docker stop: Detiene un contenedor en ejecución.
  • docker rm: Elimina un contenedor.
  • docker rmi: Elimina una imagen.
  • docker build: Construye una imagen a partir de un Dockerfile.
  • docker-compose up: Levanta los servicios definidos en un archivo docker-compose.yml.
  • docker-compose down: Detiene y elimina los servicios y redes creados por docker-compose up.

5. Estructura de un Proyecto Docker

Para crear un proyecto Docker, es recomendable seguir una estructura de directorios organizada. A continuación se muestra un ejemplo:


    mi_proyecto/
    ├── Makefile
    ├── .env
    └── srcs/
        ├── docker-compose.yml
        ├── mariadb/
        │   ├── Dockerfile
        │   └── ...
        ├── nginx/
        │   ├── Dockerfile
        │   └── ...
        ├── wordpress/
        │   ├── Dockerfile
        │   └── ...

El archivo Makefile debe contener las instrucciones necesarias para construir y ejecutar tu proyecto utilizando Docker.

6. Creación de un Dockerfile

Un Dockerfile es un archivo de texto que contiene las instrucciones para construir una imagen Docker. Aquí tienes un ejemplo básico para un servicio de NGINX:


    # Usar la imagen base de NGINX
    FROM nginx:alpine
    
    # Copiar archivos de configuración
    COPY ./nginx.conf /etc/nginx/nginx.conf
    
    # Exponer el puerto 443
    EXPOSE 443

7. Usando Docker Compose

Docker Compose te permite definir y ejecutar aplicaciones que constan de múltiples contenedores. Un archivo docker-compose.yml puede parecerse a esto:


    version: '3.7'
    
    services:
      nginx:
        build:
          context: ./nginx
        ports:
          - "443:443"
        volumes:
          - ./nginx/conf:/etc/nginx/conf.d
        restart: unless-stopped
    
      wordpress:
        build:
          context: ./wordpress
        environment:
          WORDPRESS_DB_HOST: mariadb
          WORDPRESS_DB_USER: ${MYSQL_USER}
          WORDPRESS_DB_PASSWORD: ${MYSQL_PASSWORD}
        restart: unless-stopped
    
      mariadb:
        image: mariadb:latest
        environment:
          MYSQL_ROOT_PASSWORD: ${MYSQL_ROOT_PASSWORD}
          MYSQL_DATABASE: wordpress
          MYSQL_USER: ${MYSQL_USER}
          MYSQL_PASSWORD: ${MYSQL_PASSWORD}
        volumes:
          - db_data:/var/lib/mysql
        restart: unless-stopped
    
    volumes:
      db_data:

  1. Construye y levanta los servicios usando:
    2. docker-compose up --build
  2. Accede a tu aplicación a través del navegador en https://localhost.

9. Buenas Prácticas

  • Evita almacenar contraseñas y secretos directamente en tus Dockerfile o en tu código fuente.
  • Utiliza un archivo .env para almacenar variables de entorno, como contraseñas y configuraciones específicas.
  • Utiliza volúmenes para persistir datos en contenedores, como bases de datos y archivos de aplicación.
  • Configura tus contenedores para que se reinicien automáticamente en caso de fallo.
  • Usa etiquetas en tus imágenes para mantener el control de versiones y facilitar la gestión de tus imágenes.
  • Comprime y optimiza tus imágenes Docker utilizando las mejores prácticas de construcción.

10. Seguridad en Docker

Es importante seguir buenas prácticas de seguridad al trabajar con Docker:

  • Siempre utiliza imágenes oficiales y de confianza.
  • Mantén Docker y sus componentes actualizados.
  • Evita ejecutar contenedores como root siempre que sea posible. Utiliza usuarios no privilegiados dentro de los contenedores.
  • Utiliza Docker secrets para manejar credenciales de forma segura.
  • Monitorea y registra la actividad de los contenedores para detectar comportamientos anómalos.

11. Configuración de Redes y Volúmenes

Docker permite la creación de redes personalizadas y el uso de volúmenes para manejar datos persistentes. Aquí tienes cómo hacerlo:


    # Crear una red personalizada
    docker network create mi_red
    
    # Crear un volumen
    docker volume create mi_volumen
    
    # Usar la red y el volumen en docker-compose.yml
    version: '3.7'
    
    services:
      mi_servicio:
        image: mi_imagen
        networks:
          - mi_red
        volumes:
          - mi_volumen:/data
    
    networks:
      mi_red:
    
    volumes:
      mi_volumen:

12. Solución de Problemas Comunes

Aquí hay algunos problemas comunes y sus soluciones al trabajar con Docker:

  • Contenedor no se inicia: Verifica los logs del contenedor con docker logs <nombre_o_id> y asegúrate de que no haya errores en la configuración.
  • Problemas de red: Asegúrate de que los contenedores estén en la misma red y que estén correctamente configurados para comunicarse entre sí.
  • Imágenes grandes: Revisa tu Dockerfile y utiliza multi-stage builds para optimizar el tamaño de las imágenes.
  • Problemas de permisos: Si experimentas problemas de permisos, considera ejecutar contenedores con un usuario no root o ajusta los permisos de los volúmenes montados.

Para el ejercicio "Inception", aquí hay un resumen de los requisitos y algunos pasos clave:

  • Implementar TLSv1.2 o TLSv1.3 en el contenedor NGINX y asegurar que el acceso se realice a través del puerto 443.

Consulta los requisitos específicos en el documento del proyecto y asegúrate de seguir todas las mejores prácticas de Docker.
